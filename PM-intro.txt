Вступ до управління проєктами.

Проєкт - сукупність робіт, що мають спільну мету.

Управління проєктами
 - діяльність з організації проєктних робіт
 - наука, яка вивчає способи, методи управління проєктами

Складові:
 - управління ресурсами 
   = матеріальними (у т.ч. грошовими)
   = людськими (HR)
   = логістика 
 - управлінням часом (Time management)
 - управління якістю
   = управління ризиками
 - управління знаннями
   = управління безпекою (інформація з обмеженим доступом)
   = bus-factor: залежність/незалежність від персоналії
   -> більш швидке/просте повторення подібного проєкту

Схеми управління прєктами (групи схем)
 - "жорсткі", "водоспадні"
 - "гнучкі", "водовирні"

Жорсткі схеми - з фіксованою послідовністю робіт на весь проєкт
Частіше за все, ця послідовність подається у вигляді діаграми Ганта

"+" зрозумілість термінів, вартості, обсягів робіт
"-" непридатність до змін ТЗ протягом виконання проєкту
=> не сприятлива для програмних проєктів, особливо, великих і довгих

Гнучкі схеми - з періодично повторюваними короткими ітераціями на
 виконання етапів проєкту
 За повторюваність схеми називають ітеративними, циклічними, спіральними

 ТЗ (на етап) -> Виконання -> Тестування -> Впровадження -> ТЗ (на етап) -> ...

 Етап (ітерація) нетривалий - 1-4 тижні

"+" - легко вносяться зміни, постійна адаптація до замовника
"-" - важко розрахувати час та вартість усього проєкту
"=" - необхідість постійного контакту з замовником та його активності

Д.З. Скласти методи класу StringHelper
- Ellipsis - обмеження рядка з додаванням ...
- UrlCombine(part1, part2) - який з'єднує частини через "/", якщо 
у частинах є слеші, то це враховується, як і їх відсутність
UrlCombine("/home/", "/index/") -> /home/index
UrlCombine("home", "index") -> /home/index

User Stories
На заміну технічному завданню (ТЗ) у гнучких схемах вживається
підхід складанні "історій користувача".
ТЗ передбачає "технічність" - цифри, алгоритми, часові ліміти тощо.
User Stories - опис бажаної функціональності з точки зору користувача
наприклад, "я хочу щоб реєстрація проходила у декілька етапів - 
замість заповнення величезної форми мені давали декілька коротких
питань-полів для заповнення."
